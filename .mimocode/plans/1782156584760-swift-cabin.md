# Feature: User-Defined LLM API Configuration

## Overview

Add a settings button + modal to the frontend that lets users configure their own LLM provider (base URL, API key, model name). Pass this config to the backend with each request. If no config is provided, fall back to the current localhost:8075 behavior.

**Branch:** `feature/User-define-API`

---

## Phase 1: Backend Changes

### Step 1.1 â€” Create `LlmConfig` record

**New file:** `CourseGuide/src/main/java/com/courseguide/dto/LlmConfig.java`

```java
package com.courseguide.dto;

public record LlmConfig(String apiBaseUrl, String apiKey, String modelName) {}
```

All fields nullable. Jackson deserializes missing fields as `null`.

### Step 1.2 â€” Modify `LlamaAnalysisService.java`

**File:** `CourseGuide/src/main/java/com/courseguide/services/LlamaAnalysisService.java`

1. **Add overloaded entry method** (keep original for backward compat):
   ```java
   public String analyzeAndGenerateCoursePlan(Map<String, Object> studentInfo, Path snapshotPdf, Path progressPdf, LlmConfig config)
   ```

2. **Modify `callLlamaApi`** to accept `LlmConfig`:
   - Use `config.apiBaseUrl()` (fallback: `"http://localhost:8075"`) to build chat URL and models URL
   - Use `config.apiKey()` (fallback: `LLAMA_API_KEY` constant)
   - If `config.modelName()` is non-blank, skip model resolution and use it directly
   - Strip trailing slashes from base URL

3. **Modify `resolveModelId`** to accept a dynamic `modelsUrl` parameter instead of using the `LLAMA_MODELS_URL` constant

4. **Remove `resolvedModelId` caching** â€” make it local to the method so different configs don't share stale cache

## Phase 2: Frontend Changes

### Step 2.1 â€” Add `ApiConfig` type, state, and localStorage persistence

**File:** `CourseGuide/frontend/src/App.tsx`

```typescript
type ApiConfig = { apiBaseUrl: string; apiKey: string; modelName: string }

const [apiConfig, setApiConfig] = useState<ApiConfig>(() => {
  try {
    const saved = localStorage.getItem('courseguide-api-config')
    return saved ? JSON.parse(saved) : { apiBaseUrl: '', apiKey: '', modelName: '' }
  } catch { return { apiBaseUrl: '', apiKey: '', modelName: '' } }
})
const [showSettings, setShowSettings] = useState(false)

useEffect(() => {
  localStorage.setItem('courseguide-api-config', JSON.stringify(apiConfig))
}, [apiConfig])
```

### Step 2.2 â€” Add gear button to header

Replace the header with a flex layout containing the title on the left and a gear icon button on the right that sets `showSettings(true)`.

### Step 2.3 â€” Add `SettingsModal` component

Inside `App()`, add a `SettingsModal` function component that renders:
- **API Base URL** input (placeholder: `http://localhost:8075`)
- **API Key** input (type: password)
- **Model Name** input (optional, placeholder: `gpt-4o, llama-3.3-70b-versatile`)
- Helper text: "Stored in your browser only. Never sent to our server."
- **Reset to Default** button (clears config + localStorage)
- **Done** button (closes modal)
- Backdrop click and X button to close

Render `{showSettings && <SettingsModal />}` in the JSX return.

### Step 2.4 â€” Pass `llmConfig` in API request

In the `submit()` function, add `llmConfig` to the request body:

```typescript
llmConfig: {
  apiBaseUrl: apiConfig.apiBaseUrl || undefined,
  apiKey: apiConfig.apiKey || undefined,
  modelName: apiConfig.modelName || undefined,
}
```

Using `|| undefined` ensures empty strings are omitted from JSON.

### Step 2.5 â€” Update terminal log

Replace the hardcoded API URL log line with a dynamic one based on `apiConfig.apiBaseUrl`.
